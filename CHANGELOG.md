## 2.0.2.2-Linnea - 2026-04-08

### Added
- 离线缓存新增“全部视频 / 文件夹”双视图，支持本地文件夹管理、多对多归类、批量添加到文件夹，以及按当前文件夹顺序播放与连播，解决缓存内容越来越多后难以整理和查找的问题。
- 离线缓存支持手动排序，并补齐文件夹排序能力，让用户可以按照自己的观看习惯组织缓存内容，而不再只能依赖默认顺序。
- 下载合集内容时会自动创建并复用对应文件夹，避免同一合集的缓存视频散落在总列表里，降低后续整理成本。
- 动态流与推荐流新增共享白名单，并支持在视频菜单和作者面板中一键加白，减少用户反复屏蔽或放行同一位 UP 主的操作成本。
- Android 新增预测性返回支持（须设置过渡动画为Native），并提供开关项，让系统返回预览动画与应用页面切换更加一致。
- “我的”页面新增类似收藏区的历史记录卡片预览，方便用户快速继续最近看过的内容。
- 搜索结果列表新增“已点击视频标题变灰”效果，帮助用户快速区分已看与未看的搜索结果。
- Android 更新体验优化，支持自动识别设备架构并匹配更合适的 APK 下载入口，更新不再需要手动选择下载。
- （上游改动）用户主页新增更多空间页能力，包括预约按钮、舰长信息展示、舰长榜入口，以及 Android 用户快捷方式支持，方便高频访问常看的 UP 主。

### Changed
- 离线缓存卡片的删除、更新弹幕、添加到文件夹等操作统一移入右下角菜单，让长按专门用于进入多选，交互方式与收藏、稍后再看更一致。
- 快速收藏相关提示文案进行了调整，更加直观。
- 视频菜单、动态作者面板以及“允许与其他应用同时播放”等部分文案与操作顺序进行了微调，减少误解并提升可发现性。
- 章节进度条显示方式进一步优化，支持更直观地展示当前章节名称与章节分割点，降低用户在长视频中定位章节的成本。
- 升级了 GetX 相关实现，替换为自己的Fork版本，为预测性返回、提供底层支持。

### Fixed
- 修复离线缓存播放从小窗展开时出现灰屏、无响应，提升 PiP 与缓存播放之间的切换稳定性。
- 修复“从小窗展开后进入听视频再返回”时播放器消失的问题，避免播放器状态恢复不完整导致页面空白。
- 修复多账号配置下无法正确恢复上次观看进度的问题，避免用户被错误定位到开头或无法继续上次播放位置。
- 修复 API 签名中空格编码不一致导致的搜索 `API key error`，提高搜索请求的兼容性与稳定性。
- 修复预测性返回相关的界面闪烁与视频页顶部白线问题，让返回动画与页面渲染更加平滑。
